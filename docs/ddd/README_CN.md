# 领域模型

此文件夹包含RuView中每个主要子系统的领域驱动设计(DDD)规范。

DDD围绕要解决的问题组织代码库——而不是围绕技术层。每个*有界上下文*拥有自己的数据、规则和语言。上下文通过领域事件通信,而不是通过共享可变状态。这使得系统更容易推理、测试和扩展——无论您是人还是AI代理。

## 如何阅读这些

每个模型定义:

- **通用语言**——在代码和对话中使用的具有精确含义的术语
- **有界上下文**——具有明确职责和边界的独立子系统
- **聚合**——强制执行业务规则的对象集群(例如,PoseTrack拥有其关键点)
- **值对象**——具有含义的不可变数据(例如,CoherenceScore不仅仅是一个浮点数)
- **领域事件**——已发生且其他上下文可能关心的事情
- **不变量**——必须始终为真的规则(例如,"漂移警报需要>3天的>2sigma")
- **反腐败层**——在不泄露内部信息的情况下在上下文之间转换的适配器
