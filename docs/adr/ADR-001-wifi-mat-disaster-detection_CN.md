# ADR-001: WiFi-Mat灾难检测架构

## 状态
已接受

## 日期
2026-01-13

## 背景

地震、建筑物倒塌、雪崩和洪水等自然灾害将受害者困在瓦砾或废墟下。传统的搜索和救援方法使用目视检查、热成像相机或声学设备,存在显著限制:

- **视觉/光学**:无法穿透瓦砾、废墟或倒塌的结构
- **热成像**:穿透深度有限,受环境温度影响
- **声学**:需要受害者发出声音,误报率高
- **K9单位**:可用性有限,疲劳,环境危害

基于WiFi的感知提供了独特优势:**射频信号可以穿透非金属废墟**(混凝土、木材、干墙)并通过信道状态信息(CSI)分析检测细微的人体运动,包括呼吸模式和心跳。

### 问题陈述

我们需要对WiFi-DensePose Rust实现的模块化扩展,要求:

1. 在灾难场景中高灵敏度地检测人体存在
2. 在瓦砾/废墟场中定位幸存者
3. 分类受害者状态(有意识运动、仅呼吸、危急)
4. 向救援团队提供实时警报
5. 使用便携硬件在降级/现场条件下运行

## 决策

我们将创建一个新的crate `wifi-densepose-mat`(大规模伤亡评估工具)作为现有Rust workspace的模块化添加,采用以下架构:

### 1. 领域驱动设计(DDD)方法

该模块遵循DDD原则,具有清晰的有界上下文:

```
wifi-densepose-mat/
├── src/
│   ├── domain/           # 核心领域实体和值对象
│   │   ├── survivor.rs   # 带状态跟踪的幸存者实体
│   │   ├── disaster_event.rs  # 灾难事件聚合根
│   │   ├── scan_zone.rs  # 正在扫描的地理区域
│   │   └── alert.rs      # 警报值对象
│   ├── detection/        # 生命体征检测有界上下文
│   │   ├── breathing.rs  # 呼吸模式检测
│   │   ├── heartbeat.rs  # 微多普勒心跳检测
│   │   ├── movement.rs   # 粗细运动分类
│   │   └── classifier.rs # 多模态受害者分类器
│   ├── localization/     # 位置估计有界上下文
│   │   ├── triangulation.rs  # 多AP三角测量
│   │   ├── fingerprinting.rs # CSI指纹匹配
│   │   └── depth.rs      # 瓦砾中的深度/层估计
│   ├── alerting/         # 通知有界上下文
│   │   ├── priority.rs   # 检伤分类优先级计算
│   │   ├── dispatcher.rs # 警报路由和调度
│   │   └── protocols.rs  # 紧急协议集成
│   └── integration/      # 反腐败层
│       ├── signal_adapter.rs  # 适配wifi-densepose-signal
│       └── nn_adapter.rs      # 适配wifi-densepose-nn
```

### 2. 核心架构决策

#### 2.1 事件驱动架构
- 所有幸存者检测发出领域事件
- 事件支持审计跟踪和重放,用于事后分析
- 支持多个扫描团队的分布式部署

#### 2.2 可配置检测管道
```rust
pub struct DetectionPipeline {
    breathing_detector: BreathingDetector,
    heartbeat_detector: HeartbeatDetector,
    movement_classifier: MovementClassifier,
    ensemble_classifier: EnsembleClassifier,
}
```

#### 2.3 检伤分类(START协议兼容)
| 状态 | 检测标准 | 优先级 |
|--------|-------------------|----------|
| 立即(红) | 检测到呼吸,无运动 | P1 |
| 延迟(黄) | 运动 + 呼吸,稳定生命体征 | P2 |
| 轻微(绿) | 强运动,响应模式 | P3 |
| 死亡(黑) | 连续扫描>30分钟无生命体征 | P4 |

#### 2.4 硬件抽象
支持多种部署场景:
- **便携式**:单TX/RX配手持设备
- **分布式**:在倒塌现场周围部署多个AP
- **无人机搭载**:用于大区域的基于UAV扫描
- **车载**:带阵列的移动指挥所

### 4. 性能要求

| 指标 | 目标 | 理由 |
|--------|--------|-----------|
| 检测延迟 | <500ms | 救援人员实时反馈 |
| 误报率 | <5% | 最小化浪费的救援工作 |
| 漏报率 | <1% | 不能错过幸存者 |
| 穿透深度 | 3-5m | 典型瓦砾堆深度 |
| 电池寿命(便携式) | >8小时 | 全班次运行 |
| 并发区域 | 16+ | 大型灾难现场覆盖 |

### 5. 安全和可靠性

- **故障安全默认**:对模糊信号始终假设存在生命
- **冗余检测**:多种算法对存在进行投票
- **持续监控**:定期重新扫描区域
- **离线运行**:无需网络即可完全运行
- **审计日志**:所有检测的完整跟踪

## 后果

### 积极影响
- 模块化设计允许独立开发和测试
- DDD确保领域专家可以验证逻辑
- 事件驱动支持分布式部署
- 适配器隔离上游更改
- 与现有WiFi-DensePose基础设施兼容

### 消极影响
- 事件系统增加的复杂性
- 救援团队的学习曲线
- 需要针对不同瓦砾类型进行校准
- 灾难区域中的射频干扰

### 风险和缓解措施
| 风险 | 缓解措施 |
|------|------------|
| 金属废墟阻挡信号 | 多角度扫描,自适应频率 |
| 环境射频干扰 | 频谱感知,跳频 |
| 动物误报 | 大小/模式分类 |
| 现场电源限制 | 低功耗模式,太阳能充电 |
