# ADR-014: WiFi感知的最先进信号处理算法

## 状态
已接受

## 背景

现有信号处理管道(ADR-002)提供了基础CSI处理:相位解绕、基于FFT的特征提取和基于方差的运动检测。然而,WiFi感知的学术最先进技术(2020-2025)已显著超越这些基础。为了实现研究级精度,我们需要基于WiFi信号传播和人体交互物理学的算法。

### 当前差距与SOTA对比

| 能力 | 当前 | SOTA参考 |
|-----------|---------|----------------|
| 相位清理 | Z-score异常值 + 解绕 | 共轭乘法(SpotFi 2015, IndoTrack 2017) |
| 异常值检测 | Z-score | Hampel滤波器(基于鲁棒中位数) |
| 呼吸检测 | 过零频率 | 菲涅尔区模型(FarSense 2019, Wi-Sleep 2021) |
| 信号表示 | 原始幅位/相位 | CSI频谱图(时频2D矩阵) |
| 子载波使用 | 所有子载波同等 | 基于敏感度的选择(方差比) |
| 运动分析 | 单一运动分数 | 身体速度分布/BVP(Widar 3.0 2019) |

## 决策

在`wifi-densepose-signal` crate中实现六个SOTA算法作为新模块,每个都有确定性测试且无模拟数据。

### 1. 共轭乘法(CSI比率模型)

**是什么**:将来自天线对(i,j)的CSI乘以`H_i * conj(H_j)`以取消载波频率偏移(CFO)、采样频率偏移(SFO)和包检测延迟——所有这些都破坏原始相位测量。

**为什么**:来自消费级硬件(ESP32、Intel 5300)的原始CSI相位包含每包变化的随机偏移。共轭乘法仅保留由环境(人体运动)引起的相位差异,而非硬件。

**数学**: `CSI_ratio[k] = H_1[k] * conj(H_2[k])`,其中k是子载波索引。
结果相位`angle(CSI_ratio[k])`仅反映两个天线元素之间的路径差异。

**参考**: SpotFi (SIGCOMM 2015), IndoTrack (MobiCom 2017)

### 2. Hampel滤波器

**是什么**:使用运行中位数±缩放MAD(中位数绝对偏差)替换异常值,这本身对异常值具有鲁棒性(不同于均值/标准差Z-score)。

**为什么**:WiFi CSI具有突发干扰、多径峰值和硬件故障,这些会产生异常值。Z-score异常值检测使用均值/标准差,而它们本身就被异常值破坏(掩蔽效应)。Hampel滤波器使用中位数/MAD,可抵抗高达50%的污染。

**数学**:对于样本i周围的窗口:`median = med(x[i-w..i+w])`,
`MAD = med(|x[j] - median|)`, `σ_est = 1.4826 * MAD`。
如果`|x[i] - median| > t * σ_est`,则用中位数替换x[i]。

**参考**:标准DSP技术,用于WiGest (2015), WiDance (2017)

### 3. 菲涅尔区呼吸模型

**是什么**:将WiFi信号变化建模为人体胸部位移穿过菲涅尔区边界的函数。胸部在呼吸期间移动~5-10mm,这在5 GHz (λ=60mm)下是菲涅尔区宽度的显著部分。

**为什么**:过零计数对强信号有效,但在多径丰富的环境中失败。菲涅尔模型根据TX-RX-人体几何预测信号周期中的呼吸运动*应该*出现在哪里,即使信号微弱也能实现检测。

**数学**:点P处的菲涅尔区半径:`F_n = sqrt(n * λ * d1 * d2 / (d1 + d2))`。
信号变化:`ΔΦ = 2π * 2Δd / λ`,其中Δd是胸部位移。
预期呼吸幅位:`A = |sin(ΔΦ/2)|`。

**参考**: FarSense (MobiCom 2019), Wi-Sleep (UbiComp 2021)

### 4. CSI频谱图

**是什么**:通过对每个子载波的时域CSI幅位流应用滑动窗口FFT(STFT)来构建2D时频矩阵。这揭示了身体运动的频率内容如何随时间变化。

**为什么**:频谱图是基于CNN的活动识别的标准输入。呼吸的人显示~0.2-0.4 Hz频带,行走显示1-2 Hz,静止环境仅显示噪声。2D结构允许1D特征错过的空间模式识别。

**数学**:`S[t,f] = |Σ_n x[n] * w[n-t] * exp(-j2πfn)|²`

**参考**:自2018年以来几乎在所有基于CNN的WiFi感知论文中使用

### 5. 子载波敏感度选择

**是什么**:根据子载波对人体运动的敏感度(运动和静态期间的方差比)对子载波进行排名,并仅选择前K个进行进一步处理。

**为什么**:并非所有子载波对人体运动的响应都相同。有些位于多径零点,有些主要携带噪声。使用所有子载波会稀释信号。选择10-20个最敏感的子载波可将SNR提高6-10 dB。

**数学**:`sensitivity[k] = var_motion(amp[k]) / (var_static(amp[k]) + ε)`。
按敏感度分数选择前K个子载波。

**参考**: WiDance (MobiCom 2017), WiGest (SenSys 2015)

### 6. 身体速度分布(BVP)

**是什么**:从跨子载波的多普勒频移中提取身体部位的速度分布。BVP是一个2D表示(速度 × 时间),编码了不同身体部位以不同速度运动的方式。

**为什么**:BVP与域无关——无论房间布局、家具或AP放置如何,都会出现相同的速度分布。这使其成为跨环境手势和活动识别的基础。

**数学**:对每个子载波在时间上应用DFT,然后在子载波上聚合:`BVP[v,t] = Σ_k |STFT_k[v,t]|`,其中v通过`v = f_doppler * λ / 2`映射到速度。

**参考**: Widar 3.0 (MobiSys 2019), WiDar (MobiSys 2017)

## 实现

所有算法在`wifi-densepose-signal/src/`中作为新模块实现:
- `csi_ratio.rs` — 共轭乘法
- `hampel.rs` — Hampel滤波器
- `fresnel.rs` — 菲涅尔区呼吸模型
- `spectrogram.rs` — CSI频谱图生成
- `subcarrier_selection.rs` — 基于敏感度的选择
- `bvp.rs` — 身体速度分布提取

每个模块都有:
- 具有已知输入/输出的确定性单元测试
- 无随机数据,无模拟
- 带有源论文引用的文档
- 与现有`CsiData`类型的集成

## 后果

### 积极影响
- 匹配2019-2023年出版物的研究级信号处理
- 基于物理学的算法(菲涅尔区、多普勒)而不仅仅是启发式
- 通过BVP和CSI比率实现跨环境鲁棒性
- 通过频谱图提供CNN就绪特征
- 通过子载波选择提高SNR

### 消极影响
- 增加的计算成本(STFT、每帧复杂乘法)
- 菲涅尔模型需要TX-RX距离估计(几何输入)
- BVP需要足够的时域历史(100+ Hz采样时>1秒)
