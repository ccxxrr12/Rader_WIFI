## 模块

### DTW 手势学习 (`lrn_dtw_gesture_learn.rs`)

**功能**：通过执行 3 次来教学设备自定义手势。它最多记住 16 种不同的手势。当它识别到您教给它的手势时，它会触发带有手势 ID 的事件。

**算法**：具有 3 次排练注册协议的动态时间规整（DTW）。

DTW 测量两个可能在速度上变化的时间序列之间的相似性。与简单的相关性不同，DTW 可以将缓慢执行的手势与快速执行的手势匹配。Sakoe-Chiba 带（宽度=8）约束规整路径以防止病理匹配。

#### 学习协议

```
  状态机：

  Idle ──(60 帧静止)──> WaitingStill
    ^                                 |
    |                            (检测到运动)
    |                                 v
    |                             Recording ──(静止)──> Captured
    |                                                           |
    |                                                    (保存排练)
    |                                                           |
    |                                          +----- < 3 次排练? ──> WaitingStill
    |                                          |
    |                                     >= 3 次排练
    |                                          |
    |                                   (检查 DTW 相似性)
    |                                          |
    +-- (所有 3 个相似?) ──> 提交模板 ──+
    +-- (差异太大?) ──> 丢弃并重置 ──+
```

#### 教程：教学您的 ESP32 自定义手势

**步骤 1：进入训练模式。**
站立静止 3 秒钟（20 Hz 时 60 帧）。设备检测到持续静止并进入 `WaitingStill` 模式。基础固件中没有 LED 指示器，但您可以通过监听状态转换来添加一个。

**步骤 2：执行手势。**
在 WiFi 场中移动您的手。设备记录相位增量轨迹。录制最多捕获 64 个样本（20 Hz 时 3.2 秒）。保持手势在 3 秒以内。

**步骤 3：返回静止。**
停止移动。设备将录制捕获为「3 次排练中的第 1 次」。

**步骤 4：再重复 2 次。**
设备保持在学习模式。再执行相同的手势两次，每次后返回静止。

**步骤 5：自动验证。**
第 3 次排练后，设备计算所有 3 次录制之间的成对 DTW 距离。如果所有 3 个都相互相似（DTW 距离 < 3.0），它会将它们平均为一个模板并分配手势 ID 100（第一个自定义手势）。后续手势获得 ID 101、102 等。

**步骤 6：识别。**
一旦存储了模板，设备会持续将传入的相位增量流与所有存储的模板进行匹配。当找到匹配（DTW 距离 < 2.5）时，它会发出 `GESTURE_MATCHED` 及其手势 ID，并进入 2 秒冷却期以防止双重触发。

**可靠手势识别的提示：**
- 在房间的相同大致区域执行手势
- 使手势独特（挥手比缓慢挥手更容易与圆圈区分）
- 训练期间避免环境运动（其他人行走、风扇）
- 较短的手势（0.5-1.5 秒）往往比长手势更可靠

### 异常吸引子 (`lrn_anomaly_attractor.rs`)

**功能**：将房间的 WiFi 信号建模为动态系统并对其行为进行分类。空房间产生「点吸引子」（稳定信号）。有 HVAC 的房间产生「极限环」（周期性）。有人的房间产生「奇怪吸引子」（复杂但有界）。当信号离开学习的吸引子盆地时，说明发生了不寻常的事情。

**算法**：带有 Lyapunov 指数估计的 4D 动态系统分析。

状态向量为：`(mean_phase, mean_amplitude, variance, motion_energy)`

Lyapunov 指数量化轨迹发散：
```
lambda = (1/N) * sum(log(|delta_n+1| / |delta_n|))
```
- lambda < -0.01: **点吸引子**（稳定，空房间）
- -0.01 <= lambda < 0.01: **极限环**（周期性，机器/HVAC）
- lambda >= 0.01: **奇怪吸引子**（混沌，有人的房间）

学习 200 帧后（约 10 秒），吸引子类型被分类并建立盆地半径。后续超出 3 倍盆地半径的偏离会触发异常警报。

```rust
    pub fn attractor_type() -> AttractorType;    // Unknown/PointAttractor/LimitCycle/StrangeAttractor
    pub fn is_initialized() -> bool;             // 200 学习帧后为 True
}

```

#### 教程：理解吸引子类型

**点吸引子 (lambda < -0.01)**
信号收敛到固定点。这意味着环境完全静态 -- 没有人，没有机器，没有气流。WiFi 信号是确定性且不变的。任何扰动都会触发盆地偏离。

**极限环 (lambda 接近 0)**
信号遵循周期性轨道。这通常表示机械系统：HVAC 循环、风扇、电梯机械。周期通常与设备的工作周期匹配。极限环上的人类活动会将 Lyapunov 指数推为正值。

**奇怪吸引子 (lambda > 0.01)**
信号有界但非周期性 -- 经典混沌。这是人类活动的特征：行走、手势、呼吸都会产生复杂但有界的信号动态。人越多，Lyapunov 指数往往越高。

**盆地偏离**
盆地偏离意味着当前信号状态距离吸引子中心超过 3 倍学习半径。这可能表示：
- 有人新进入房间
- 门或窗户打开
- 设备开启/关闭
- 环境变化（下雨、温度）

### 元适应 (`lrn_meta_adapt.rs`)

**功能**：自动调整 8 个检测阈值以减少误报并提高检测准确性。使用真实世界反馈（真阳性和假阳性）驱动简单的爬山优化器。

**算法**：带安全回滚的迭代参数扰动。

优化器维护 8 个参数，每个参数都有边界和步长：

| 索引 | 参数 | 默认值 | 范围 | 步长 |
|------|------|--------|------|------|
| 0 | 存在阈值 | 0.05 | 0.01-0.50 | 0.01 |
| 1 | 运动阈值 | 0.10 | 0.02-1.00 | 0.02 |
| 2 | 相干性阈值 | 0.70 | 0.30-0.99 | 0.02 |
| 3 | 手势 DTW 阈值 | 2.50 | 0.50-5.00 | 0.20 |
| 4 | 异常能量比率 | 50.0 | 10.0-200.0 | 5.0 |
| 5 | 区域占用阈值 | 0.02 | 0.005-0.10 | 0.005 |
| 6 | 生命体征呼吸暂停秒数 | 20.0 | 10.0-60.0 | 2.0 |
| 7 | 入侵敏感度 | 0.30 | 0.05-0.90 | 0.03 |

优化循环（在定时器上运行，非每帧）：
1. 测量基线性能分数：`score = TP_rate - 2 * FP_rate`
2. 将一个参数按其步长扰动（交替 +/- 方向）
3. 等待 `EVAL_WINDOW`（10）个定时器 tick
4. 测量新的性能分数
5. 如果改进，保留更改。如果没有，恢复。
6. 3 次连续失败后，安全回滚到最后一个已知良好的快照。
7. 扫过所有 8 个参数，然后增加元级别计数器。

假阳性的 2 倍惩罚反映了现实世界的成本：假警报（因为系统认为检测到运动而在凌晨 3 点叫醒某人）比偶尔错过真实事件更糟糕。

#### 教程：向元适应提供反馈

元适配器需要反馈来了解其更改是否有帮助。在典型部署中：

1. **真阳性**：当事件（存在检测、手势匹配）被另一个传感器或用户确认正确时，调用 `report_true_positive()`。
2. **假阳性**：当事件触发但实际上没有发生任何事情时（例如，在空房间中检测到存在），调用 `report_false_positive()`。
3. **通用事件**：为所有事件调用 `report_event()`，无论正确性如何，以归一化分数。

在没有人类反馈的自主操作中，您可以使用模块之间的交叉验证：如果相干性门和异常吸引子都同意发生了某事，则将其视为真阳性。如果只有一个触发，则可能是假阳性。

### EWC 终身学习 (`lrn_ewc_lifelong.rs`)

**功能**：使用 WiFi 信号特征学习对人所在的区域进行分类（最多 4 个区域）。关键是，当移动到新环境时，它学习新布局而不会忘记以前学习的布局。这是由弹性权重巩固实现的「终身学习」属性。

**算法**：在 8 输入、4 输出线性分类器上的 EWC（Kirkpatrick et al., 2017）。

分类器有 32 个可学习参数（8 个输入 x 4 个输出）。训练使用带有 EWC 惩罚项的梯度下降：

```
L_total = L_current + (lambda/2) * sum_i(F_i * (theta_i - theta_i*)^2)
```

- `L_current` = 预测区域与独热目标之间的 MSE
- `F_i` = Fisher 信息对角线（每个参数对先前任务的重要性）
- `theta_i*` = 先前任务结束时的参数值
- `lambda` = 1000（强正则化以防止遗忘）

梯度通过有限差分估计（将每个参数扰动 epsilon=0.01，测量损失变化）。每帧只更新 4 个参数（轮询）以保持在 2ms 预算内。

#### 任务边界检测

「任务」对应于稳定环境（房间布局）。任务边界自动检测：
1. 跟踪损失 < 0.1 的连续帧
2. 100 连续稳定帧后，提交任务：
   - 快照参数为 `theta_star`
   - 从累积梯度平方更新 Fisher 对角线
   - 重置稳定性计数器

Fisher 内存饱和前最多可学习 32 个任务。

#### 教程：微控制器上的终身学习如何工作

**问题**：传统神经网络遭受「灾难性遗忘」。如果您在房间 A 上训练网络，然后在房间 B 上训练它，它会忘记关于房间 A 的一切。这是一个基本限制，不是 bug。

**EWC 解决方案**：在学习房间 B 之前，系统测量哪些参数对房间 A 很重要（通过 Fisher 信息对角线）。然后，在学习房间 B 时，它添加一个惩罚，防止对房间 A 重要的参数变化太大。结果：网络学习房间 B，同时保留房间 A 的知识。

**在 ESP32 上**：分类器故意很小（32 个参数），以将计算保持在每帧 2ms 以内。尽管其简单性，对 8 个子载波组特征的线性分类器可以可靠地区分 4 个空间区域。Fisher 对角线每个任务仅需要 32 个浮点数（128 字节）。最多 32 个任务，总 Fisher 内存约为 4 KB。

**监控遗忘风险**：`FORGETTING_RISK` 事件（ID 748）报告 EWC 惩罚与当前损失的比率。如果此比率超过 1.0，EWC 约束正在主导学习信号，意味着系统在学习新任务时难以不忘记旧任务。这可能发生在：
- 新环境与所有先前环境非常不同
- 32 参数模型容量耗尽
- Fisher 对角线因任务过多而饱和

## 微控制器上的学习如何工作

ESP32-S3 约束塑造所有自适应学习模块的设计：

### 无 GPU
所有计算在 CPU（240 MHz 的 Xtensa LX7 双核）上通过 WASM3 解释器完成。这意味着：
- 无矩阵乘法硬件
- 无并行 SIMD 操作
- 每个浮点操作都很重要

### 固定内存
WASM3 分配固定的线性内存区域。没有堆，没有 `malloc`，没有动态分配：
- 所有数组都是固定大小且栈分配的
- 最大数据结构大小是编译时常量
- 缓冲区溢出不可能（Rust 的边界检查 + 固定数组）

### 防止遗忘的 EWC
没有 EWC，将设备移动到新房间会擦除所有关于先前房间的学习。EWC 每个任务增加 ~32 个浮点数的开销（Fisher 对角线快照），这在 ESP32 上可以忽略不计。

### 轮询梯度估计
每帧计算所有 32 个参数的梯度会花费太长时间。相反，EWC 模块使用轮询调度：每帧 4 个参数，在 8 帧内循环所有 32 个。在 20 Hz 时，完整的梯度传递需要 0.4 秒 -- 对于房间占用的慢动态来说足够快。

### 任务边界检测
系统自动检测何时「收敛」到新环境（100 连续稳定帧 = 5 秒的一致低损失）。无需手动干预。用户只需将设备放置在新房间，学习会自动发生。

### 能量预算

| 模块 | 预算 | 每帧操作 | 内存 |
|------|------|----------|------|
| DTW 手势学习 | H (<10ms) | DTW: 64x64=4096 次乘法/模板，最多 16 个模板 | ~18 KB (模板 + 排练) |
| 异常吸引子 | S (<5ms) | 4D 距离 + Lyapunov 的 log + EMA | ~2.5 KB (128 轨迹点) |
| 元适应 | S (<5ms) | 分数计算 + 扰动（仅定时器，非每帧） | ~256 字节 |
| EWC 终身学习 | L (<2ms) | 4 次有限差分评估 + 梯度步 | ~512 字节 (参数 + Fisher + theta_star) |

所有 4 个学习模块的总静态内存：约 21 KB。
