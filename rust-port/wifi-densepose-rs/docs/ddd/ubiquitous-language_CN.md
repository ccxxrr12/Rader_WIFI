# 通用语言

本词汇表定义了 WiFi-DensePose 系统中使用的领域术语。所有团队成员（开发人员、领域专家、利益相关者）应在代码、文档和对话中一致使用这些术语。

---

## 核心概念

### WiFi-DensePose

使用 WiFi 信号执行非侵入式人体姿态估计的系统。与基于摄像头的系统不同，它可以穿透墙壁并在黑暗中运行，提供保护隐私的身体追踪。

### 信道状态信息（CSI）

关于 WiFi 信号如何在发射器和接收器之间传播的详细信息。CSI 捕获多个子载波和天线上的幅度和相位变化，编码包括人体存在和运动在内的环境信息。

### DensePose

一种计算机视觉技术，将检测到的人体的所有像素映射到 3D 表面表示。在我们的上下文中，我们将 WiFi 信号转换为与 DensePose 兼容的输出。

### 姿态估计

确定人体位置和方向的过程，通常通过识别解剖学标志点（关键点）和身体部位来实现。

## 信号领域术语

### 幅度

特定子载波和天线对的 CSI 测量的大小（强度）。幅度变化指示环境中的物理变化，特别是人体运动。

**单位：** 线性刻度或分贝（dB）

**使用示例：**
```rust
let amplitude = csi_frame.amplitude(); // 幅度值矩阵
```

### 相位

WiFi 信号的时间偏移，以弧度为单位。相位对距离变化高度敏感，对于检测呼吸等微妙运动至关重要。

**单位：** 弧度（-pi 到 pi）

**注意：** 原始相位在使用前需要净化（解缠、噪声去除）。

### 子载波

OFDM（正交频分复用）WiFi 信号中的单个频率分量。每个子载波提供信道状态的独立测量。

**典型值：**
- 20 MHz 带宽：56 个子载波
- 40 MHz 带宽：114 个子载波
- 80 MHz 带宽：242 个子载波

### 天线

WiFi 设备上的物理接收器元件。多个天线启用 MIMO（多输入多输出）并在 CSI 测量中提供空间多样性。

**典型配置：** 1x1、2x2、3x3、4x4

### 信噪比（SNR）

衡量所需信号相对于背景噪声强度的质量指标。更高的 SNR 表示更干净、更可靠的 CSI 数据。

**单位：** 分贝（dB）

**质量阈值：**
- SNR < 10 dB：质量差，可能无法使用
- SNR 10-20 dB：可接受的质量
- SNR > 20 dB：良好的质量

### 噪声底

环境中的电磁干扰水平。噪声底限制了最小可检测信号。

**单位：** dBm（相对于毫瓦的分贝）

### 多普勒频移

由移动物体引起的频率变化。CSI 中的多普勒效应揭示了运动速度和方向。

**公式：** fd = (2 * v * f) / c

其中 v 是速度，f 是载波频率，c 是光速。

### 功率谱密度（PSD）

信号功率在频率上的分布。PSD 分析揭示了步行或呼吸等周期性运动。

**单位：** dB/Hz

### 特征提取

从原始 CSI 数据计算有意义的统计信息和转换的过程。特征包括幅度均值/方差、相位差、相关性和频域特征。

### 预处理

初始信号调节包括：
- **噪声去除** - 过滤掉低质量测量
- **加窗** - 应用窗函数（汉明、汉恩）以减少频谱泄漏
- **归一化** - 将值缩放到标准范围
- **相位净化** - 解缠和平滑相位数据

## 姿态领域术语

### 模态转换

WiFi-DensePose 的核心创新：将射频（RF）特征转换为可由姿态估计算模型处理的类视觉特征表示。

**也称为：** 跨模态学习，射频到视觉转换

### 人体存在检测

确定一个或多个人是否存在于感应区域的二元分类。这通常是姿态估计管道的第一阶段。

### 人数统计

检测区域中估计的个体数量。由于信号叠加，使用 WiFi 感应进行准确计数具有挑战性。

| 2 | 右眼 | 右眼中心 |

| 4 | 右耳 | 右耳 |
| 5 | 左肩 | 左肩关节 |
| 6 | 右肩 | 右肩关节 |
| 7 | 左肘 | 左肘关节 |
| 8 | 右肘 | 右肘关节 |

### UV 坐标

身体表面的 2D 参数化。U 和 V 是连续坐标（0-1），将身体上的任何点映射到规范 3D 网格。

**目的：** 无论姿态如何，都能实现一致的身体表面表示。

### 边界框

检测空间中包围检测到的人的矩形区域。

**格式：** (x, y, width, height) 在归一化坐标 [0, 1] 中

### 置信度分数

指示模型对检测或分类的确定性的概率值 [0, 1]。值越高表示信心越大。

**阈值：**
- 低：< 0.5
- 中：0.5 - 0.8
- 高：> 0.8

### 活动

对人正在做什么的高级分类：

| 活动 | 描述 |
|----------|-------------|
| 站立 | 直立，静止 |
| 坐着 | 坐姿 |
| 行走 | 步行运动 |
| 跑步 | 快速步行运动 |
| 躺下 | 水平位置 |
| 跌倒 | 快速过渡到地面 |
| 未知 | 未分类的活动 |

### 跌倒检测

识别跌倒事件，通常特征为：
1. 快速垂直速度
2. 水平最终位置
3. 突然减速（撞击）
4. 随后的静止

**关键用例：** 老年人护理，医疗设施

### 运动检测

识别感应区域中的显著运动。通过以下方式检测运动：
- CSI 幅度/相位变化
- 多普勒频移分析
- 时间特征变化

## 流领域术语

### 会话

用于实时数据流式传输的客户端连接。会话有生命周期：连接中、活动、暂停、重新连接、完成、失败。

### 流类型

正在流式传输的数据类别：

| 类型 | 数据内容 |
|------|--------------|
| 姿态 | 姿态估计结果 |
| CSI | 原始或处理后的 CSI 数据 |
| 警报 | 关键事件（跌倒、运动） |
| 状态 | 系统健康和指标 |

### 区域

用于过滤和组织检测的逻辑或物理区域。区域启用：
- 单个系统覆盖多个房间
- 按区域订阅
- 位置感知警报

### 订阅

客户端表示对接收特定数据的兴趣。订阅包括：
- 流类型
- 区域过滤器
- 置信度阈值
- 节流偏好

### 广播

向所有匹配订阅条件的客户端发送数据。

### 心跳

用于验证连接活跃性的定期 ping 消息。未能响应心跳的客户端将被断开连接。

### 背压

当客户端无法足够快地处理消息时的流控制机制。选项包括：
- 缓冲（有限）
- 丢弃帧
- 节流源

### 延迟

事件发生到客户端接收之间的时间延迟。以毫秒为单位测量。

**目标：** 实时应用 < 100ms

## 硬件领域术语

### 设备

能够提取 CSI 的物理 WiFi 硬件单元。支持的类型：

| 类型 | 描述 |
|------|-------------|
| ESP32 | 带 WiFi 的低成本微控制器 |
| Atheros 路由器 | 带有修改固件的路由器 |
| Intel NIC | Intel 5300/5500 网络卡 |
| Nexmon | 带有 Nexmon 固件的 Broadcom 芯片 |
| PicoScenes | 研究级 CSI 平台 |

### MAC 地址

媒体访问控制地址 - 网络接口的唯一硬件标识符。

**格式：** XX:XX:XX:XX:XX:XX（十六进制）

### 固件

在 WiFi 设备上运行的启用 CSI 提取的软件。

### 校准

调整设备以获得最佳 CSI 质量的过程：
1. 测量噪声底
2. 计算天线相位偏移
3. 建立基线信号特征

### 健康检查

定期验证设备是否正常运行。检查包括：
- 连接性
- 数据速率
- 错误率
- 温度（如果可用）

## 存储领域术语

### 仓库

用于持久化和检索聚合根的接口。每种聚合类型都有自己的仓库。

**模式：** 领域驱动设计中的仓库模式

### 实体

具有随时间持续存在的独特标识的对象。如果实体的标识符匹配，则它们相等。

**示例：** 设备、会话、CsiFrame

### 值对象

由其属性而非标识定义的对象。值对象是不可变的，如果所有属性匹配，则它们相等。

**示例：** 频率、置信度、MacAddress

### 聚合

被视为单个单元的实体和值对象的集群。一个实体是聚合根；所有访问都通过它进行。

### 事件存储

将领域事件存储为事实来源的持久化机制。支持事件溯源和审计跟踪。

## 跨领域术语

### 限界上下文

特定领域模型在其中定义和适用的逻辑边界。每个限界上下文都有自己的通用语言。

**WiFi-DensePose 上下文：**
1. 信号（CSI 处理）
2. 姿态（推断）
3. 流（实时交付）
4. 存储（持久化）
5. 硬件（设备管理）

### 领域事件

域中发生的重要事情的记录。事件是不可变的，以过去时态命名。

**示例：** CsiFrameReceived、PoseEstimated、FallDetected

### 命令

执行可能更改系统状态的操作的请求。

**示例：** ProcessCsiFrame、EstimatePose、ConnectDevice

### 查询

请求信息但不更改状态的请求。

**示例：** GetCurrentPose、GetDeviceStatus、GetSessionHistory

### 关联 ID

链接系统中相关事件的唯一标识符，实现端到端跟踪。

## 指标和质量术语

### 吞吐量

数据处理速率，通常以以下方式测量：
- CSI 的每秒帧数（FPS）
- 推断的每秒姿态数
- 流式传输的每秒消息数

### 处理时间

完成处理步骤的持续时间。以毫秒为单位测量。

### 准确性

估计值与地面实况的接近程度。对于姿态估计：
- 关键点的 OKS（对象关键点相似度）
- 边界框的 IoU（交并比）

### 精确率

正确的正检测的比例。

**公式：** TP / (TP + FP)

### 召回率

检测到的实际阳性的比例。

**公式：** TP / (TP + FN)

### F1 分数

精确率和召回率的调和平均值。

**公式：** 2 * (精确率 * 召回率) / (精确率 + 召回率)

## 使用指南

### 在代码中

使用本词汇表中的精确术语：

```rust
// 良好：使用通用语言
pub struct CsiFrame { ... }
pub fn detect_human_presence(&self) -> HumanPresenceResult { ... }
pub fn estimate_pose(&self) -> PoseEstimate { ... }

// 不良：非标准术语
pub struct WifiData { ... }  // 应为 CsiFrame
pub fn find_people(&self) { ... }  // 应为 detect_human_presence
pub fn get_body_position(&self) { ... }  // 应为 estimate_pose
```

### 在文档中

始终使用定义的术语；避免可能引起混淆的同义词。

### 在对话中

讨论系统时，始终使用这些术语，以确保技术专家和领域专家之间的清晰沟通。
